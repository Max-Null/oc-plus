# 双星 V1 — 仲裁器 + 左右脑 + 执行器

> 状态：❌ 已废弃 | 归档于 2026-07-15

---

## 架构

```
双星仲裁器 (primary, temp 0.1, readonly)
  │  接收任务 → 并行调用左右脑 → 仲裁整合 → 下发指令给执行器
  │
  ├─ 左脑 (subagent, temp 0.2, readonly)
  │    微观路径规划——从当前状态出发，贪心推进下一步
  │    输出 JSON：role, confidence, current_status, next_immediate_action, blockers
  │
  ├─ 右脑 (subagent, temp 0.7, readonly)
  │    宏观目标定位——从目标出发校准方向
  │    输出 JSON：role, confidence, target_alignment, course_correction, milestone_check
  │
  └─ 构建执行器 (subagent, temp 0.1, edit+bash)
       接收仲裁器指令 → 严格执行 → 报告结果
       铁律：不参与规划、不质疑指令、只执行
```

## 核心机制

### 仲裁规则

- 左右脑输出一致 → 直接采纳
- 冲突时：
  - 右脑 target_alignment ≥ 20% 偏差 → 右脑优先
  - 偏差 < 20% → 左脑优先
- 置信度处理：
  - 双方 confidence < 0.5 → 不生成指令，请求更多上下文
  - 单方 confidence < 0.3 → 忽略该侧
  - confidence 0.3-0.5 → 采纳但标记

### JSON 容错

1. 优先 json.loads() 解析
2. 失败则正则提取关键字段
3. 两次提取均失败 → 记录日志，请求用户手动确认
4. 左右脑各自最多重试 2 次，每次温度 +0.1

## 设计意图

- 双路并行理解（左脑微观 + 右脑宏观），互补决策盲区
- 仲裁器不亲自思考——只做拼接
- 执行器权限分离——思考者只读，执行者有写权限

## 废弃原因

- 仲裁器只是"拼接者"，不对问题做自己的深度理解
- 左右脑必须输出 JSON 结构化数据，限制了 LLM 的自然推理能力
- 只适用于编码任务，无法处理文档/研究/规划类任务
- 执行器是"盲执行者"——只接指令，不理解上下文
