# 双星 V2 — 主力智能助手 + 参谋/军师顾问

> 状态：✅ 当前基线（已被 V3 增强）

---

## 架构

```
双星 (primary, temp 0.2, edit+bash)
  │  智能助手，先判断任务类型，再选择策略
  │
  ├─ 非编码 / 简单编码 → 自己理解 + 自己干
  │
  └─ 关键决策时 → 并行咨询：
        ├─ 参谋 (subagent, temp 0.7, readonly)
        │    战术纠偏——从目标锚定角度审视方向是否偏离
        │
        └─ 军师 (subagent, temp 0.3, readonly, glob+grep)
             战略远见——从全局上下文审视方案的全局最优性
```

## 核心变化（相对 V1）

### 删掉的

| 删除 | 原因 |
|------|------|
| 旧双星仲裁器 | 只是拼接者，不对问题做自己的深度理解 |
| 左脑 | JSON 结构化输出限制了自然推理 |
| 右脑 | 同上 |
| 构建执行器 | 双星自己拥有 edit+bash，不需要独立执行器 |

### 改的

| 变化 | 说明 |
|------|------|
| 双星从"仲裁器"改为"智能助手" | 去掉角色锁死，恢复模型的自然多面性 |
| 双星拥有 edit+bash 权限 | 自己能做的事不委托 |
| 参谋从"宏观目标"改为"战术纠偏" | 不再是 JSON 输出，改为自然语言目标偏差检查 |
| 军师从"无"改为"战略远见" | 四策略（编码/文档/研究/规划），主动探索代码库做全局审视 |
| 顾问调用从"每轮必调"改为"仅关键决策" | 节省 thinking 预算，简单任务不启动顾问 |

## 设计意图

- CC 式的"薄 prompt"——不给模型预设角色，让上下文驱动行为
- 规范写进能力，不写进身份——灵活应变
- 参谋做目标锚定（防止过度设计），军师做全局审视（防止代码孤岛）

## 已知局限

- 缺少专用编码 subagent（复杂编码全部双星自己干，缺乏系统流程）
- 缺少 LSP 级别的代码智能（依赖 grep/read 猜测代码关系）
- 缺少用户意图提炼（搜索时直接搬运用户原词）
