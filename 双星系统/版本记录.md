## 版本号使用规则

1. **主版本号（V3）** 在以下情况升级：架构变更（新增/删除 agent）、核心流程重构（如引入四阶段）
2. **功能编号（.1）** 在以下情况升级：prompt 微调、bug 修复、权限调整、文档更新
3. 版本号写入 `设计.md` 的变更历史表
4. 每次改动机通过 git commit 记录，commit message 格式：`feat(双星): V3.1 — 修复军师调用时机`
