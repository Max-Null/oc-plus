## 触发线 4 ASSERTION_RE 的盲区

**事实**：`ASSERTION_RE` 当前正则抓的是显性断言句型——`不支持|做不到|只有 N 种|不存在|无法|远低于|过于`。抓不到比较级/隐性推断句。

**漏网句型**：
- ❌ 已被抓：「OC 不支持 plan mode」
- 🔴 漏网：「OC 的 edit 比 CC 的更可靠」——形式是推断，实质同样是未查证的结论，但句型绕过了正则
- 🔴 漏网：「相比 X 我更信任 Y」「这个方案更简洁」——不依赖断言骨架词

**反例证明**：实际对话中 LLM 对 CC/OC edit 差异的判断纯属猜测，被用户揭穿后才承认。触发线 4 未触发。

**结论**：断言正则覆盖的是冰山露出水面的部分（显性否定句/穷举句）。比较级推断句是水下部分，只能靠用户的直接反馈。不扩展正则——扩展会误伤正常比较分析。保持现状，依赖分形的 plan 文档机制和用户把关来兜底。

**原则**：承认触发线 4 是筛子不是墙，它的价值在截最蠢的硬伤，不是拦截所有 LLM 自我欺骗。剩下的交给 plan 审阅和用户判断。
