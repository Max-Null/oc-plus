**事实**：oc-plus 项目的文档目录结构：doc/ 下有四个子目录——原型/（产品原型）、设计/（功能方案）、计划/（排期里程碑）、知识/（业务架构参考）。

**原则**：各子目录必须有 README.md 做索引；进入新项目或不确定项目结构时，先 read README.md 了解目录布局和模块用途。

**结论**：写文档时按内容类型放入对应子目录；新项目先读 README.md。
